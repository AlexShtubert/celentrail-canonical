# 00_MASTER_CONTEXT — что это за репозиторий

Это репозиторий системы Celentrail.
Цель: хранить **в одном месте** и в **максимально машиночитаемом виде**:

- каноническое описание системы (текст)
- актуальные 3D-модели (GLB/STEP и т.п.)
- параметры (JSON) для автоматического расчёта/симуляции
- историю изменений (changelog)
- стандарты именования и структуры

## Единая “вечная” ссылка
Ссылка на этот репозиторий — это и есть “вечная” ссылка.
Внутри репозитория всегда есть папка `assets/.../latest/`, которую ИИ трактует как источник истины.

## Правило обновления
Любое изменение через ИИ -> ИИ генерирует новые файлы и обновляет `docs/` + кладёт модель в `assets/.../versions/<версия>/`
и обновляет `assets/.../latest/` на новую версию.

## Как дать любому ИИ задачу
Дай ссылку на репозиторий и скажи:
- “Открой docs/00_MASTER_CONTEXT.md и docs/10_BALKA.md, затем работай строго по ним. Обнови latest при изменениях.”
